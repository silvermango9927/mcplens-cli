Two ways to sharpen your MCP surface

audit-mcp

Audit tool activation

Run a local audit against a tools/list export and optional usage logs. The report flags what stops agents from picking the right tool.

  • Low-discoverability tools
  • Vague or overlapping descriptions
  • Core / admin / contextual profile boundaries
  • Safer naming and description rewrites
  • Missing instrumentation to debug failures
compile → build

OpenAPI → lean MCP server

Turn a large API surface into a smaller, agent-oriented MCP server with slimmer responses — then generate a standalone TypeScript server you can ship.

  • Editable manifest of tools, requests & response maps
  • Hides UI / admin / webhook endpoints from agents
  • Strips response cruft for ~80% smaller payloads
  • Standalone server: stdio, HTTP, Docker Compose
  • Optional one-time LLM curation, or fully offline

Local and offline by design.

The audit path makes no network requests, no LLM calls, and sends no telemetry. It reads only the files you pass in and writes only the paths you name. Your tool definitions, logs, and prompts stay on your machine.

Quick start

Requires Node.js 20 or newer. No install needed with npx.

Audit an MCP server
npx mcplens-cli audit-mcp \
  --tools-list tools.json \
  --logs events.jsonl \
  --out report.md
Compile an OpenAPI spec into a lean MCP server
npx mcplens-cli compile \
  --spec openapi.json \
  --out agentify.manifest.json --offline

npx mcplens-cli build \
  --manifest agentify.manifest.json \
  --out ./my-mcp

Prefer a global install? npm install -g mcplens-cli, then run mcplens --help.
